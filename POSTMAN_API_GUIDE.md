# SpringBoot 用户模块 Postman 接口联调指南

## 项目基本信息
- **项目名称**: SpringBoot 用户管理模块
- **服务地址**: http://localhost:8080
- **基础路径**: /api/users

## 接口列表

### 1. 创建用户 (POST)
**请求信息**
- **请求方式**: POST
- **请求地址**: `http://localhost:8080/api/users`
- **Content-Type**: `application/json`

**请求体示例**
```json
{
    "id": 1,
    "name": "张三",
    "age": 25
}
```

**参数说明**
- `id`: 用户ID（可选，不传则系统自动生成）
- `name`: 用户名（必填）
- `age`: 用户年龄（必填）

**成功响应示例**
```json
{
    "code": 200,
    "msg": "创建用户成功，用户名：张三，年龄：25",
    "data": {
        "id": 1,
        "name": "张三",
        "age": 25
    }
}
```

**验证标准**
- 状态码: 200
- code字段: 200
- msg字段: 包含"创建用户成功"和用户名、年龄信息
- data字段: 包含完整的用户信息

### 2. 查询用户 (GET)
**请求信息**
- **请求方式**: GET
- **请求地址**: `http://localhost:8080/api/users/{id}`
- **路径参数**: {id} 替换为实际用户ID

**请求示例**
- 查询ID为1的用户: `http://localhost:8080/api/users/1`

**成功响应示例**
```json
{
    "code": 200,
    "msg": "查询用户成功，用户ID：1",
    "data": {
        "id": 1,
        "name": "张三",
        "age": 25
    }
}
```

**错误响应示例**（用户不存在）
```json
{
    "code": 404,
    "msg": "用户不存在，用户ID：999",
    "data": null
}
```

**验证标准**
- 状态码: 200（成功）或 404（用户不存在）
- code字段: 200（成功）或 404（用户不存在）
- msg字段: 包含相应的提示信息
- data字段: 成功时包含用户信息，失败时为null

### 3. 更新用户 (PUT)
**请求信息**
- **请求方式**: PUT
- **请求地址**: `http://localhost:8080/api/users/{id}`
- **Content-Type**: `application/json`
- **路径参数**: {id} 替换为要更新的用户ID

**请求体示例**
```json
{
    "name": "李四",
    "age": 30
}
```

**请求示例**
- 更新ID为1的用户: `http://localhost:8080/api/users/1`

**成功响应示例**
```json
{
    "code": 200,
    "msg": "更新用户成功，用户ID：1，用户名：李四",
    "data": {
        "id": 1,
        "name": "李四",
        "age": 30
    }
}
```

**验证标准**
- 状态码: 200
- code字段: 200
- msg字段: 包含"更新用户成功"和用户ID、用户名信息
- data字段: 包含更新后的用户信息

### 4. 删除用户 (DELETE)
**请求信息**
- **请求方式**: DELETE
- **请求地址**: `http://localhost:8080/api/users/{id}`
- **路径参数**: {id} 替换为要删除的用户ID

**请求示例**
- 删除ID为1的用户: `http://localhost:8080/api/users/1`

**成功响应示例**
```json
{
    "code": 200,
    "msg": "删除用户成功，用户ID：1",
    "data": null
}
```

**验证标准**
- 状态码: 200
- code字段: 200
- msg字段: 包含"删除用户成功"和用户ID信息
- data字段: null

### 5. 获取所有用户列表 (GET)
**请求信息**
- **请求方式**: GET
- **请求地址**: `http://localhost:8080/api/users`

**成功响应示例**
```json
{
    "code": 200,
    "msg": "获取用户列表成功",
    "data": {
        "1": {
            "id": 1,
            "name": "张三",
            "age": 25
```

## Postman 操作步骤

### 环境准备
1. 确保SpringBoot应用已启动（端口8080）
2. 打开Postman应用程序
3. 创建新的Collection（可选）

### 接口测试顺序建议
1. **创建用户** → 2. **查询用户** → 3. **更新用户** → 4. **删除用户**

### 详细操作步骤

#### 创建用户 (POST)
1. 在Postman中新建请求
2. 选择请求方式: **POST**
3. 输入URL: `http://localhost:8080/api/users`
4. 点击"Headers"标签，添加:
   - Key: `Content-Type`
   - Value: `application/json`
5. 点击"Body"标签，选择"raw"，格式选择"JSON"
6. 输入JSON请求体
7. 点击"Send"发送请求
8. 查看响应结果

#### 查询用户 (GET)
1. 新建请求
2. 选择请求方式: **GET**
3. 输入URL: `http://localhost:8080/api/users/1`（替换1为实际ID）
4. 直接点击"Send"发送请求
5. 查看响应结果

#### 更新用户 (PUT)
1. 新建请求
2. 选择请求方式: **PUT**
3. 输入URL: `http://localhost:8080/api/users/1`（替换1为实际ID）
4. 设置Headers（同POST请求）
5. 输入JSON请求体
6. 点击"Send"发送请求
7. 查看响应结果

#### 删除用户 (DELETE)
1. 新建请求
2. 选择请求方式: **DELETE**
3. 输入URL: `http://localhost:8080/api/users/1`（替换1为实际ID）
4. 直接点击"Send"发送请求
5. 查看响应结果

## 常见错误处理

### 400 Bad Request
- **原因**: JSON格式错误、参数缺失、参数类型错误
- **解决方案**: 检查JSON格式，确保参数完整且类型正确

### 404 Not Found
- **原因**: 用户不存在、URL路径错误
- **解决方案**: 检查用户ID是否正确，确认URL路径

### 500 Internal Server Error
- **原因**: 服务器内部异常
- **解决方案**: 查看服务器日志，检查代码逻辑

## 测试用例建议

### 正常流程测试
1. 创建用户 → 查询用户 → 更新用户 → 删除用户
2. 创建多个用户 → 查询用户列表

### 异常流程测试
1. 查询不存在的用户（验证404响应）
2. 创建重复ID的用户（验证400响应）
3. 发送格式错误的JSON（验证400响应）
4. 更新不存在的用户（验证404响应）

## 注意事项

1. **ID生成规则**: 如果不传ID，系统会自动生成基于时间戳的ID
2. **数据持久化**: 当前使用内存存储，重启应用数据会丢失
3. **并发安全**: 当前版本未处理并发问题，适合单用户测试
4. **参数验证**: 系统会对参数进行基本验证，返回相应的错误信息

通过以上步骤，您可以完整地测试SpringBoot用户模块的所有接口功能。
